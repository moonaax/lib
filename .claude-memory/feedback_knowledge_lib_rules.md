向 `/Users/mi/lib/my-knowledge-lib/` 下写入任何 `.md` 文件前,必须先执行以下三步:

1. **定性** — 明确文件性质:
   - **正式笔记**(学习类文档)→ 严格遵守 CLAUDE.md 的"文档规范"12 条
   - **过程文件**(待办/备忘/索引/TODO)→ 放入 `_待整理/` 或类似下划线开头的辅助目录,在文件开头标注性质,不套用正式笔记规范
2. **对照规范**:一级标题 / `>` 引言 / `[[双链]]` / `---` 分隔 / 编号章节 / 深度内容 / 代码语言标注 / 🔑 重点 / 3000+ 字 / 双链校验 / `## 面试题精选` / 查重
3. **写完自检** — 按 12 条清单复核,特别是双链有效性和面试题章节

**Why:** 用户在 2026-04-29 明确指出:我多次给知识库加文档时没主动对照 CLAUDE.md,导致创建了一份既不符合正式笔记规范、又没标注为过程文件的"两头不靠"文档(`待整理-实战知识点.md`)。CLAUDE.md 虽然在 project instructions 里自动加载,但"存在于上下文"不等于"激活执行",必须主动对照清单。

**How to apply:** 任何 `Write`/`Edit` 操作目标路径在 `/Users/mi/lib/my-knowledge-lib/` 下且是 `.md` 文件时,先停下来走上述三步。如果无法判断是正式笔记还是过程文件,向用户确认而不是自作主张。
